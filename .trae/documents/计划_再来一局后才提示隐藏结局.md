# 计划：再来一局后才提示隐藏结局

## Summary
- 目标：结局页不再自动弹出“检测到你持有…是否进入隐藏结局？”；改为玩家点击“再来一局”时，若持有信物才弹出该提示。
- 预期结果：玩家先完整看到四种常规结局内容；点击“再来一局”后才决定是否进入隐藏结局。

## Current State Analysis
- 结局页实现集中在 [game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js) 的 `screens.ending`。
- 当前信物提示为“自动触发”：
  - 在 `ending.update()` 中，当 `mode === 'normal'`、`Game.state.relic` 存在且淡入完成度较高时，将 `mode` 切到 `relicPrompt`，从而直接弹窗。
  - 该行为导致结局页一进入就弹窗，不符合“先看到常规结局，点击再来一局再提示”的需求。
- “再来一局”当前行为为“直接重开”：
  - `ending.onClick('replay')` 直接 `Game.reset()` 并 `Game.switchScreen('gacha')`。

## Decisions（已确认）
- 点击“再来一局”后弹出信物提示时，选择“不进入”应返回结局页（关闭弹窗），并且后续再次点击“再来一局”直接开新一局。
- 当已经进入隐藏结局（或隐藏分支失败提示）时，“再来一局”保持现状：直接开新一局，不再额外弹信物提示。

## Proposed Changes
### 1) 取消结局页自动弹窗触发
- 文件：[game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js)
- 位置：`screens.ending.update()`
- 修改：
  - 移除/禁用以下自动切换逻辑：当 `Game.state.relic` 存在且 `endingAlpha > 0.95` 时自动 `s.mode = 'relicPrompt'`。
- 影响：
  - 进入结局页时始终展示常规结局内容（以及“再来一局”按钮），不再自动弹窗。

### 2) 将“信物提示”改为由“再来一局”触发
- 文件：[game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js)
- 位置：`screens.ending.onClick(id)`
- 修改（仅对 `id === 'replay'` 分支加条件）：
  - 当满足以下条件时，不重开游戏，而是弹窗：
    - `s.mode === 'normal'`
    - `!s.relicHandled`
    - `Game.state.relic` 存在
  - 动作为：`s.mode = 'relicPrompt'` 并 `return`（阻止 `reset + switchScreen`）。
  - 其他情况（例如 `mode === 'hidden'`）保持现有逻辑：点击“再来一局”直接 `reset` 并跳转到 `gacha`。
- 与既有弹窗按钮行为的衔接：
  - “不进入”：现有逻辑会将 `mode` 设回 `normal` 并 `relicHandled = true`，从而回到结局页并允许再次点击“再来一局”直接重开。
  - “进入”：沿用现有 `relicBranch -> hidden/relicFail` 流程，不做改动。

## Verification
- 手动验证（浏览器/H5）：
  - Case A：无信物通关 → 结局页不弹窗 → 点击“再来一局”直接进入抽卡/新开局流程。
  - Case B：有信物通关 → 进入结局页不弹窗 → 点击“再来一局”弹出“检测到你持有…” → 点“不进入”回到结局页 → 再点“再来一局”直接进入抽卡/新开局流程。
  - Case C：有信物通关 → 点击“再来一局”弹窗 → 点“进入”并完成分支进入隐藏结局 → 在隐藏结局页点“再来一局”直接进入抽卡/新开局流程。
  - Case D：有信物通关 → 点击“再来一局”弹窗 → 点“进入”但分支失败进入“副本未开启”提示 → 点“返回结局”回到常规结局页 → 点“再来一局”直接开新一局（不再弹窗）。

## Notes / Non-goals
- 不改动隐藏结局分支规则、隐藏结局文案与解锁逻辑，仅调整触发时机（从“进入结局页自动触发”改为“点击再来一局触发”）。
