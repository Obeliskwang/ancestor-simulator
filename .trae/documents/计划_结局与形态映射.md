# 计划：结局与物种形态一一对应（按 4 档进度）

## 摘要
将 **结局档位（族群衰落/勉强存续/一方豪强/地球霸主）** 与每个种族的 **4 段进化链形态名** 一一对应，并在结局页展示“最终形态名称”，例如海洋线：
- 族群衰落 → 头甲鱼（孱弱古鱼）
- 勉强存续 → 初始全骨鱼
- 一方豪强 → 粒骨鱼
- 地球霸主 → 巨型邓氏鱼

同理适用于刃齿虎与孔子鸟两条进化链。

## 现状分析（基于代码）
- 种族数据已包含 `evolutionChain`（4 个形态名）与 `name`（用于括号内的“孱弱古鱼/羸弱原兽/无翼古雏”）。见：[data.js](file:///e:/Obeliskw/ancestor-simulator/js/data.js#L2-L38)
- 结局档位已固定为 4 档（💀/🌱/⚔️/🏆），结局配置在 `GameData.endings`，结局页渲染取 `ending.name/description/formText`。见：[data.js](file:///e:/Obeliskw/ancestor-simulator/js/data.js#L139-L144)、[game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js#L1265-L1349)
- 当前结局页只显示种族名称（race.name）与结局信息，尚未显示“最终形态名”。

## 目标与验收标准
### 目标
- 结局页在生物图标/种族名附近新增一行：`最终形态：{形态名}`，形态名由“结局档位”映射到该种族的进化链。
- 形态名映射规则（四档）：
  - 💀 族群衰落（progress < 30）→ evolutionChain[0] + `（${race.name}）`
  - 🌱 勉强存续（30~59）→ evolutionChain[1]
  - ⚔️ 一方豪强（60~89）→ evolutionChain[2]
  - 🏆 地球霸主（90~100）→ evolutionChain[3]

### 验收
- 选择任意种族通关后：
  - progress < 30：显示 “{初始形态}（{race.name}）”
  - 30~59：显示进化链第 2 段
  - 60~89：显示进化链第 3 段
  - 90~100：显示进化链第 4 段
- 三个种族都符合各自 `evolutionChain`：
  - 海洋：头甲鱼/初始全骨鱼/粒骨鱼/巨型邓氏鱼
  - 陆地：细齿兽/伪剑齿虎/纤细刃齿虎/毁灭刃齿虎
  - 天际：中华龙鸟/赫氏近鸟龙/始祖鸟/圣贤孔子鸟

## 拟修改内容（具体到文件）
### 1) 增加“结局档位 → 形态索引”的统一函数
**文件**：[data.js](file:///e:/Obeliskw/ancestor-simulator/js/data.js)
- 新增一个纯函数（挂在 `GameData` 上）：
  - `getEndingTierIndex(progress)` → 0/1/2/3
  - 或 `getFormNameForProgress(race, progress)` → string
- 这样结局页与未来其他页面需要展示形态时可复用，避免在多个 render 里复制 if/else。

### 2) 结局页展示“最终形态”
**文件**：[game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js)
- 在结局页 `race.name` 下方增加一行文本：
  - `最终形态：${formName}`
- `formName` 的生成：
  - `const tier = GameData.getEndingTierIndex(Game.state.progress)`
  - `const base = race.evolutionChain[tier]`
  - `tier===0 ? `${base}（${race.name}）` : base`
- 若 `race` 缺失或 `evolutionChain` 不存在，则不渲染该行（保持兜底稳定性）。

## 验证步骤
1) 跑通任意一局进入结局页，确认出现“最终形态：xxx”。
2) 通过正常游玩或临时改进度值分别验证 4 档显示是否正确。
3) 切换三种种族各跑一局，确认形态名与各自进化链一致。
