# 计划：隐藏结局延后触发与“信物”系统（结局后询问进入副本）

## 摘要
把 PRD 奇遇的“隐藏副本”触发时机从【奇遇阶段当场追问】改为【四个常规结局结算后追问】：
- 奇遇阶段若选择隐藏选项 A，则获得对应“信物”（仅记录，不当场进入副本）。
- 结局页显示常规结局后，如果检测到持有信物，弹出提示：“检测到你持有某信物…是否进入隐藏结局？”
- 选择“是”进入副本选择（A/B）；选对则解锁并展示隐藏结局（界面结构仿照结局页）；选错则提示“很遗憾，目标占领进度不足，副本未开启”，并回到常规结局界面。

## 目标与验收标准
### 目标
1) 奇遇隐藏选项 A 只获得信物，不当场进入副本、不当场出现 branchQuestion。
2) 常规结局页出现后，若本局持有信物，则出现“进入隐藏结局？”二次确认弹窗：
   - 彗星降临：星碎（伴随神秘呢喃）
   - 神秘种子：不知名种子（内里却涌动着暗沉浊光）
   - 远古秘境：甲骨牌（能指引秘境方向）
3) 奇遇阶段的即时反馈文案按你的补充调整：
   - 选隐藏选项：弹出提示框说明“获得信物”，且信物名带 emoji（例如：`获得信物：☄️ 星碎`）
   - 选常规选项：弹出提示框显示 PRD 5.4 的常规选项 B 对应结果文案（例如：`平安无事` / `毫无收获` 等），而不是只显示“进度 +5%”
4) 选择“是”后进入副本选项 A/B：
   - 选对（按 encounter.branchOptionA.hidden=true 视为正确）→ 解锁并展示对应隐藏结局
   - 选错 → 提示“很遗憾，目标占领进度不足，副本未开启”，回到常规结局界面
5) 隐藏结局展示页结构仿照结局页（大标题 + 图标 + 文案框 + 最终进度 + 再来一局），但标题/配色以隐藏结局为主（紫色系即可）。

### 验收
- 奇遇阶段选隐藏 A：仅出现“获得信物/进度变化”的提示，然后照常进入天灾；不会出现 branchQuestion。
- 进入结局页后：
  - 若未选隐藏 A：不弹窗。
  - 若选过隐藏 A：弹窗显示信物名与描述，并提供“进入/不进入”。
  - 选择进入 → 出现副本 A/B 选择；A 成功展示隐藏结局；B 显示失败提示并回到常规结局页。
- 本局成功解锁隐藏结局后，localStorage 的 `unlockedEndings` 会新增对应隐藏结局 id（stellar/invasive/spiritual）。

## 设计与数据约定（决定项）
### 信物定义（新增运行时状态，不做持久化）
- 新增 `Game.state.relic`（或 `pendingRelic`）用于记录本局是否持有信物：
  - `relic = { encounterId, trigger, name, description }`
  - `trigger` 使用 `encounter.hiddenOption.branch`（comet_branch_a / seed_branch_a / cave_branch_a）

### 隐藏结局解锁规则
- 成功条件：玩家在结局页副本选择中选中 **branchOptionA**（即 `encounter.branchOptionA.hidden === true`）。
- 解锁映射：用 `trigger` 去 `GameData.hiddenEndings` 中匹配 `hiddenEndings[].trigger === trigger`，得到隐藏结局对象（包含 `id/name/description`）。
- 将 `Game.state.hiddenEnding` 统一改为 **隐藏结局 id（stellar/invasive/spiritual）**，修复目前 trigger/id 混用导致的匹配问题。

## 拟修改内容（具体到文件）

**文件**：[game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js)

- `encounter.onClick`：

    - 显示结果提示（例如 `获得信物：☄️ 星碎`，必要时第二行显示简短描述/进度），然后按原节奏进入天灾
  - 不再在奇遇阶段写入 `Game.state.hiddenEnding`
  - 选中 `normalOption` 时：
    - 显示 PRD 5.4 常规选项 B 对应的结果文案（来自数据配置），并按原节奏进入天灾

### 2) 结局阶段：新增“信物提示 → 是否进入隐藏结局 → 副本 A/B”流程
**文件**：[game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js)
- `ending.init`：
  - 初始化新增子状态 `mode`，例如：
    - `mode: 'normal' | 'relicPrompt' | 'relicBranch' | 'relicFail' | 'hidden'`
  - 若存在 `Game.state.relic`，且本局尚未处理隐藏副本，则在 `update` 中自动切到 `relicPrompt`（或提供按钮触发）。
- `ending.render`：
  - `mode==='normal'`：渲染常规结局（保持现有布局）。
  - 在 `relicPrompt/relicBranch/relicFail/hidden` 下叠加绘制一个居中弹窗（矩形面板+文字+按钮）：
    - `relicPrompt`：显示“检测到你持有：{信物名}（{描述}）是否进入隐藏结局？”按钮：`进入` / `不进入`
    - `relicBranch`：显示 `encounter.branchQuestion`，按钮：`branchA` / `branchB`
    - `relicFail`：显示失败文案（固定：`很遗憾，目标占领进度不足，副本未开启`），按钮：`返回结局`
    - `hidden`：渲染“隐藏结局页”（用隐藏结局对象替代 ending 文案与标题色），并保留“再来一局”
  - 移除旧的 “hiddenEnding 在结局页底部展示” 那段逻辑，避免新旧流程重复。[game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js#L1325-L1342)
- `ending.onClick`：
  - 根据 `mode` 分发按钮事件：
    - `enterHidden`：切到 `relicBranch`
    - `stayNormal`：回到 `normal`，并标记 `relicHandled=true`（不再弹）
    - `branchA/branchB`：
      - A：根据 trigger 找到隐藏结局对象，写入 `Game.state.hiddenEnding = hidden.id`，并切 `mode='hidden'`；同时更新 localStorage 解锁列表
      - B：切 `mode='relicFail'`
    - `backToNormal`：切回 `normal`（且 `relicHandled=true`）

### 3) 数据：补充每个奇遇的信物信息（推荐放在 data.js）
**文件**：[data.js](file:///e:/Obeliskw/ancestor-simulator/js/data.js)

- 这样结局页弹窗能直接读取当前 encounter 的信物信息，避免在逻辑层 hardcode。

**文件**：[game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js)
