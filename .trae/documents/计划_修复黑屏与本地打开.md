# 计划：修复黑屏与本地打开说明

## 摘要
本次处理两件事：
1) 解释为什么建议用本地服务器打开，而不是直接双击 index.html。
2) 修复当前进入后黑屏的问题（实际是过渡遮罩永久覆盖画面 + 个别屏幕状态未初始化，后续也会崩）。

## 现状分析（基于代码与页面实测）
- 页面无报错，控制台仅输出“初始化完成”，但画面只剩背景渐变（截图确认为黑屏）。主因在 [game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js) 的过渡层逻辑：`transitionAlpha` 只增不减，且完成过渡后仍保持 `1`，导致每帧都用全不透明矩形盖住画面。
- 另外，`conflict`、`disaster` 等屏幕的 `screenState` 没有初始化 `questionIndex`，一旦进入这些阶段，会出现 `q = questions[undefined]` 进而报错或渲染异常。
- `Utils.updateSize()` 每次 resize 都会再次 `ctx.scale(dpr,dpr)`，未重置 transform，反复 resize 会导致坐标与缩放失真。

## 为什么建议用本地服务器打开（回答问题 1）
直接双击 `index.html` 会以 `file://` 协议打开。很多浏览器在 `file://` 下会有额外限制或不一致行为，例如：
- `localStorage` 在部分环境下会受限或表现不一致（尤其是隐私模式/严格设置）。
- 相对路径与资源加载、缓存策略在 `file://` 与 `http://` 下差异更大，排查问题不直观。
- 真实上线环境永远是 `http(s)://`，本地用 `python -m http.server` 能更接近线上运行方式，减少“本地能跑、线上不行”的概率。

结论：不是“必须”开服务器才能用，但**开服务器更稳定、也更贴近真实发布环境**。

## 黑屏原因与修复方向（回答问题 2）
### 根因
- 过渡层遮罩绘制条件：`if (isTransitioning || transitionAlpha > 0)`。
- `transitionAlpha` 在 `update()` 中从 0 增到 1 后不再回到 0，因此遮罩一直存在且全不透明，画面被永久盖住。

### 修复方案（决策已锁定）
实现“两段式过渡”：淡出旧屏 → 切屏 → 淡入新屏。并保证过渡结束后 `transitionAlpha = 0`。

## 拟修改内容（具体到文件）
### 1) 修复过渡遮罩导致黑屏
- 文件： [game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js)
- 修改点：
  - 在 `state` 中新增 `transitionPhase: 'idle' | 'fadeOut' | 'fadeIn'`（或等价实现）。
  - `switchScreen()`：先进入 `fadeOut`，在淡出完成瞬间执行 `init()` 并进入 `fadeIn`。
  - `update()`：分别推进 alpha（fadeOut: 0→1，fadeIn: 1→0），在 `fadeIn` 完成后重置到 `idle` 且 `transitionAlpha=0`。
  - `render()`：仅在 `transitionAlpha > 0` 时绘制遮罩，alpha 使用 `transitionAlpha`。

### 2) 补齐各阶段的 screenState 初始化，避免后续阶段崩溃
- 文件： [game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js)
- 修改点：
  - `conflict.init()`、`disaster.init()`：补 `questionIndex: 0`。
  - `encounter` 分支：补齐隐藏副本的 B 选项（当前只画/处理了 branchA），否则逻辑不完整（不影响当前黑屏，但会影响后续体验）。
  - `ending.render()`：将 `this.animationState` 改为 `UI.animationState`（避免按钮点击区域失效）。

### 3) 修复 Canvas resize 反复缩放问题（稳定性）
- 文件： [utils.js](file:///e:/Obeliskw/ancestor-simulator/js/utils.js)
- 修改点：
  - `updateSize()` 中在 `scale()` 前调用 `ctx.setTransform(1, 0, 0, 1, 0, 0)`，避免重复叠加缩放。
