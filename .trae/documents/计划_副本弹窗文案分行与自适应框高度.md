## 总结

在结局页“进入隐藏结局 → 副本选项（A/B）”弹窗（`ending.render` 的 `mode === 'relicBranch'`）中：

1) 将“副本问题（branchQuestion）”与“选项文字（branchOptionA/B）”按宽度自动换行分行展示。  
2) 弹窗主框高度根据分行后的文本动态计算，确保问题与两个选项都被主框完整包住，不再出现文字溢出框外的情况。

## 现状分析（基于代码）

- 副本弹窗 UI 位于 [game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js#L1390-L1482)：
  - 主框 `mw/mh` 为固定高度 `mh = h * 0.28`；

  - `branchOptionA/B.text` 同样单行绘制；
  - 当问题句子较长时会水平溢出；当文本变多时也可能超出固定高度主框。

## 改动方案（无需额外决策）

### 1) 在 ending.render 内增加“按宽度折行”小工具

**文件**： [game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js)

在 `ending.render` 的“弹窗模式”（`s.mode !== 'normal' && s.mode !== 'hidden'`）分支内，新增局部函数（不外提到全局，避免影响其他模块）：

- `wrapText(text, maxWidth, font, maxLines)`：
  - `ctx.save(); ctx.font = font; ... ctx.measureText(...) ...; ctx.restore();`
  - 逐字符累加，超宽则换行；
  - 超出 `maxLines` 截断；
  - 返回 `lines: string[]`。

该实现方式与结局描述/奇遇描述自动换行的写法一致，避免引入新依赖。

### 2) relicBranch：问题与选项分行绘制

**文件**： [game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js)

将 `mode === 'relicBranch'` 渲染改为：

- 主框内边距：
  - `paddingX = 18`
  - `paddingTop = 20`
  - `paddingBottom = 18`
  - `innerW = mw - paddingX * 2`
- **问题（branchQuestion）**：
  - `questionLines = wrapText(encounter.branchQuestion, innerW, 'bold 13px ...', 3)`
  - 按行从 `my + paddingTop + 18` 开始绘制，行高 `lineHeight = 18`；
- **选项按钮**：
  - `bw = innerW`
  - `bh = 44`（保持当前按钮高度）
  - `optionPaddingX = 14`
  - `optionInnerW = bw - optionPaddingX * 2`
  - `aLines = wrapText(branchOptionA.text, optionInnerW, 'bold 12px ...', 2)`
  - `bLines = wrapText(branchOptionB.text, optionInnerW, 'normal 12px ...', 2)`
  - 选项文本按行垂直居中绘制在按钮内部（用 `startY = centerY - ((lines.length - 1) * lineHeight)/2`）。

### 3) 主框高度自适应，保证“问题+两按钮”全部包住

**文件**： [game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js)

仅在 `mode === 'relicBranch'` 时按内容计算 `mh`：

- `questionBlockH = questionLines.length * lineHeight`
- `gap1 = 14`（问题到选项A的间距）
- `gap2 = 12`（A 到 B 的间距）
- `contentH = paddingTop + questionBlockH + gap1 + bh + gap2 + bh + paddingBottom`
- `mh = clamp(contentH, h * 0.28, h * 0.42)`（避免极端机型过大/过小）
- `my = (h - mh) / 2`（居中）

并将主框 `Utils.drawRect(mx, my, mw, mh, ...)` 与按钮 Y 坐标全部改用新的 `my/mh` 计算结果。
