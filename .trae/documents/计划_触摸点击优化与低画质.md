# 计划：触摸点击优化与低画质（先关特效）

## 摘要
解决两个问题：
1) 手机上需要点多次/像“两只手点”才生效：优化输入事件，保证单次点击/触摸必进下一步。
2) 先做“全设备低画质”：关闭背景粒子，弱化耗性能特效；按钮交互只做“按下变暗”，不做缩放。

## 现状分析（已在代码中确认）
### 1) 触摸点击不稳定的根因
- 文件：[main.js](file:///e:/Obeliskw/ancestor-simulator/js/main.js#L33-L93)
- 现在 `touchend` 里调用 `getEventPos(e)`，但 `touchend` 事件经常没有 `touches`，而代码会退回取 `e.clientX/e.clientY`（在 touchend 上可能是 `undefined`），导致坐标变成 `NaN`，最终点不到按钮区域。
- 同时触摸流程里先 `touchstart` 更新 hover，再 `touchend` 才点击；再叠加 hover 缩放效果，容易出现“第一下只是变亮/变大，第二下才点中”的错觉。

### 2) 性能与特效
- 文件：[ui.js](file:///e:/Obeliskw/ancestor-simulator/js/ui.js#L48-L72)
- `drawBackground()` 每帧都 `Math.random()` 画 50 个点，属于纯装饰但会持续耗性能（且会闪烁）。
- 多个按钮有 hover 缩放（`hoverScale`），在手机上既不必要，也会增加重绘/手感不稳。

## 目标与验收标准
### 目标
- 手机/桌面：点击“开始游戏”一次就切到抽取界面；抽取按钮一次必触发抽取。
- 全设备默认低画质：无背景粒子；按钮按下只变暗，无缩放。

### 验收
- 手机：单指轻点一次即可进入下一界面；不需要重复点击。
- 桌面 Chrome：鼠标单击稳定；无“卡在开始游戏”。
- 控制台无异常；页面帧率更稳定（肉眼观感不卡顿）。

## 方案与改动点（决策已锁定）

### A. 输入系统统一：改用 Pointer Events（优先）+ 兼容兜底
目的：用同一套事件同时覆盖鼠标与触摸，避免 click/touch 重复与坐标不一致。

**修改文件**： [main.js](file:///e:/Obeliskw/ancestor-simulator/js/main.js)
- 替换现有的 `click/touchstart/touchend/mousemove` 监听为：
  - `pointerdown`：计算坐标 → 立即触发 `Game.handleClick(x,y)`（带 200ms 防抖，覆盖所有输入类型）
  - `pointermove`：仅在 `pointerType === 'mouse'` 时更新 hover（手机不需要 hover）
- `getEventPos(e)`：统一使用 `e.clientX/e.clientY`；如遇极端浏览器不支持 pointer，则兜底使用 `changedTouches[0]`。
- 确保 `preventDefault()` 只用于触摸相关 pointer（避免页面滚动/双击缩放干扰）。

### B. UI 交互降级：按钮不再缩放，只做按下变暗
**修改文件**： [ui.js](file:///e:/Obeliskw/ancestor-simulator/js/ui.js)
- 移除所有 `hoverScale`（开始按钮、抽取按钮、确认按钮等）。
- 新增 `UI.pressedButton`（或等价状态）：
  - `pointerdown` 时记录 pressedButton
  - `pointerup/cancel` 时清空
- 渲染按钮时：
  - hovered（鼠标）：边框/发光轻微变化即可
  - pressed（鼠标/触摸）：填充颜色整体变暗（例如乘一个 0.85 系数或换深一档颜色）

### C. 默认低画质：关闭背景粒子等纯装饰
**修改文件**： [ui.js](file:///e:/Obeliskw/ancestor-simulator/js/ui.js)
- 增加 `UI.settings = { quality: 'low' }`（默认 low）
- `drawBackground()`：
  - low：只画渐变底色（不再每帧 random 粒子）
  - future（预留）：high 才画粒子（本次先不做 UI 开关）
- 额外（属于 low 画质的一部分）：
  - 降低 shadowBlur（例如从 30~40 降到 10~15）
  - 关闭进度条 `Math.random()` 闪点（避免持续随机绘制）

## 实施步骤
1) main.js：重构输入事件为 pointer 方案 + 修正坐标计算（touchend NaN 问题一次解决）。
2) ui.js：按钮交互移除缩放，加入 pressed 变暗。
3) ui.js：实现 `quality: low` 默认策略，关闭背景粒子并弱化重特效（阴影/闪点）。
4) 本地验证：
   - 桌面 Chrome：开始按钮/抽取按钮单击即响应。
   - 手机 Chrome：单指点击一次即响应，不需要多点。

## 风险与规避
- Pointer Events 兼容性：现代 Chrome/Android WebView 都支持；仍保留 touch 兜底逻辑以防万一。
- 防抖过强导致“点了没反应”：防抖限定 200ms，且只影响连续快速点，不影响正常点击。
