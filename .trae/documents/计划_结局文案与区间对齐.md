# 计划：结局文案与“地表占领进度”区间对齐（按 PRD 4.3 / 5.6）

## 摘要
将结局页展示的“结局名称/说明”与 **地表占领进度（Game.state.progress）** 的区间严格对齐，并确保文案与 PRD 的 **4.3 结局判定** 与 **5.6 结局结算模块**一致，同时补充“展示形态”信息用于结局页展示。

## 现状分析（基于仓库现状）
- 结局数据来源：`GameData.endings`。[data.js](file:///e:/Obeliskw/ancestor-simulator/js/data.js#L139-L144)
- 结局选择逻辑：`GameData.getEnding(progress)` 按 `min` 阈值从高到低匹配，逻辑正确。[data.js](file:///e:/Obeliskw/ancestor-simulator/js/data.js#L160-L167)
- 结局页渲染使用：`ending.name`、`ending.description`。[game.js](file:///e:/Obeliskw/ancestor-simulator/js/game.js#L1203-L1272)
- 与 PRD 的差异点：
  - `地球霸主`文案缺少“海洋/天空”。（当前只有“统治整片大地”）[data.js](file:///e:/Obeliskw/ancestor-simulator/js/data.js#L140)
  - PRD 4.3 表里结局名带图标（🏆/⚔️/🌱/💀）且有“展示形态”描述；当前数据未提供展示形态字段，结局页也不展示。

## 目标与验收
### 目标
- 结局名称/说明按进度区间匹配：
  - 90%~100%：🏆 地球霸主
  - 60%~89%：⚔️ 一方豪强
  - 30%~59%：🌱 勉强存续
  - <30%：💀 族群衰落
- 文案与 PRD 5.6.1 表一致（逐字对齐）。
- 结局页增加“展示形态”说明（终极霸主形态/高阶形态/基础形态/弱化形态…），避免玩家只看到名字和一段描述。

### 验收
- 将进度分别设置到 95 / 70 / 45 / 10（或通过正常游玩达到）：
  - 结局页展示对应的结局名称（含图标）与 PRD 文案；
  - “展示形态”与 PRD 4.3/5.6 的视觉表现描述一致；
  - 结局页布局不与隐藏结局区域发生遮挡。
