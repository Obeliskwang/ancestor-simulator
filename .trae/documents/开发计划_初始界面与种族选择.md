# 祖宗模拟器：万族争霸 - 开发计划（第一阶段）

## 概述
基于用户提供的PRD文档和H5开发规则，本阶段聚焦于游戏初始界面和种族选择界面的开发。

---

## 技术架构决策

### 文件结构
```
ancestor-simulator/
├── index.html              # 入口文件
├── js/
│   ├── main.js             # 主入口逻辑
│   ├── game.js             # 游戏核心逻辑
│   ├── ui.js               # UI渲染模块
│   ├── data.js             # 题库与配置数据
│   └── utils.js            # 工具函数
├── images/                 # 图片资源目录
│   ├── bg/                 # 背景图
│   ├── creatures/          # 生物形态图
│   └── ui/                 # UI元素图
└── css/
    └── style.css           # 样式文件
```

### Canvas渲染策略
- 使用单Canvas绘制游戏画面
- requestAnimationFrame驱动动画循环
- 响应式Canvas尺寸：基于viewport动态计算

## 第一阶段开发任务

### 任务1：入口界面（StartScreen）

#### 功能需求
- 显示游戏标题"祖宗模拟器：万族争霸"
- 远古粗粝风格视觉设计
- "开始游戏"按钮
- 历史最高分/已解锁结局显示（如有）
- 点击开始进入种族选择

#### 视觉设计
| 元素 | 设计要求 |
|------|----------|
| 背景 | 远古海洋/史前氛围，深色渐变背景 |
| 标题 | 火焰/血液灌注动效，史前生物写实风格 |
| 按钮 | 骨甲/岩石质感，悬停有光效 |
| 整体色调 | 土黄、深褐、血红、琥珀色系 |

#### 交互逻辑
1. 页面加载 → 显示入场动画（标题浮现）
2. 动画结束 → 显示"开始游戏"按钮
3. 点击按钮 → 淡出过渡 → 进入种族选择界面

---

### 任务2：种族抽取界面（RaceGachaScreen）

#### 功能需求
- 显示开场剧情文案
- 点击抽取按钮，随机抽取一个种族
- 抽取过程有抽卡动画效果
- 抽取结果展示种族+初始形态
- 抽取结果确认后初始化进度条（10%）

#### 种族数据（来自PRD）

| 种族ID | 名称 | 标题 | 初始形态 | 路线A | 路线B | 稀有度 |
|--------|------|------|----------|-------|-------|--------|
| dunkleosteus | 巨型邓氏鱼 | 海洋霸主 | 头甲鱼 | 防御流 | 强攻流 | 普通 |
| smilodon | 毁灭刃齿虎 | 陆地统治者 | 细齿兽 | 群居流 | 猎杀流 | 普通 |
| confuciusornis | 圣贤孔子鸟 | 天际征服者 | 中华龙鸟 | 速度流 | 强攻流 | 普通 |

#### 抽取动画设计

```
┌─────────────────────┐
│      [抽取按钮]      │  ← 中心召唤符文/祭坛
│                       │
│   3个种族图标快速轮转   │  ← 滚动动画 1.5秒
│       ↓ ↓ ↓          │
│   逐渐减速停至结果     │  ← 减速效果 1秒
│       ↓ ↓ ↓          │
│   ✨ 结果放大展示 ✨    │  ← 结果定格
│                       │
│   [开始进化] 按钮     │  ← 确认按钮
└─────────────────────┘
```

#### 视觉设计
| 元素 | 设计要求 |
|------|----------|
| 抽取按钮 | 中心祭坛/召唤阵外观，悬停发光 |
| 轮转效果 | 3个种族图标快速循环，背景模糊动感 |
| 减速动画 | 速度递减波动效果，最终停在结果 |
| 结果展示 | 金色光效边框 + 形态图 + 种族信息 |
| 稀有度特效 | 全部为普通，无特殊区分 |

#### 交互逻辑
1. 显示剧情文案（打字机效果）
2. 显示抽取祭坛/召唤阵
3. 点击"开始抽取"按钮
4. 种族图标快速轮转（1.5秒）
5. 逐渐减速（1秒），最终停在结果
6. 结果放大展示 + 形态图 + 种族名称/标题
7. 显示"开始进化"按钮
8. 点击确认 → 初始化进度条10% → 进入进化阶段

---

### 任务3：进度条组件（ProgressBar）

#### 功能需求
- 顶部固定显示
- 初始值10%
- 动态填充动画（火焰/血液灌注效果）
- 数值实时更新

#### 视觉设计
- 容器：骨甲/岩石质感边框
- 填充：从左到右渐进，颜色根据进度变化
  - 0-29%: 暗红色（衰落）
  - 30-59%: 琥珀色（存续）
  - 60-89%: 翠绿微光（豪强）
  - 90-100%: 金色光效（霸主）
- 数值显示：百分比数字+微小动画
